# Contributing to SafeRoute

Thanks for your interest in improving SafeRoute! This is a student project, so contributions of any size — bug fixes, docs, new features, or even just opening issues — are welcome.

## Getting Started

1. Fork the repository.
2. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/saferoute.git
   cd saferoute
   ```
3. Open the project in Android Studio and let Gradle sync.
4. Add your own `app/google-services.json` (see the README's Setup section) so the app can build.
5. Create a new branch for your change:
   ```bash
   git checkout -b feature/short-description
   ```

## Making Changes

- Keep changes focused — one feature or fix per pull request.
- Follow the existing Kotlin code style used in the project (standard Android/Kotlin conventions).
- Test your change on a device or emulator before submitting.
- Update the README if your change affects setup, permissions, or features.

## Submitting a Pull Request

1. Commit your changes with a clear message:
   ```bash
   git commit -m "Add: short description of the change"
   ```
2. Push to your fork:
   ```bash
   git push origin feature/short-description
   ```
3. Open a pull request against the `main` branch and describe:
   - What the change does
   - Why it's needed
   - How you tested it

## Reporting Bugs / Suggesting Features

Open a GitHub Issue with:
- A clear title
- Steps to reproduce (for bugs) or a description of the use case (for feature requests)
- Screenshots if relevant

## Code of Conduct

Be respectful and constructive. This project focuses on women's safety — please keep contributions and discussions aligned with that goal.
